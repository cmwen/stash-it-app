## Design Principles

### 1. **Invisible Saving**
The best save is one that takes zero effort. Saving should feel instant and automatic.

### 2. **Reading First**
Every design decision should optimize for comfortable, distraction-free reading.

### 3. **Offline Confidence**
Users should never wonder if something works offline. It always does.

### 4. **Progressive Disclosure**
Show basic features first, reveal advanced options when needed.

### 5. **Honest Feedback**
Always tell users what's happening - loading, syncing, errors, success.

## Accessibility Considerations

### Touch Targets
- Minimum 48x48dp for all interactive elements
- Cards: Full width, minimum 72dp height

### Screen Reader
- All images have alt text
- Buttons have descriptive labels
- Reading order follows visual hierarchy
- Announce state changes (saved, deleted, etc.)

### Visual
- Minimum 4.5:1 contrast ratio for text
- Don't rely on color alone for status
- Support system font scaling
- Reduce motion option respected

### Keyboard/Focus
- Logical tab order
- Visible focus indicators
- Escape closes modals/sheets
